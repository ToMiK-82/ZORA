# ZORA — Zootopia Operational Resource Assistant

Многоагентная система на локальных LLM для автоматизации бизнес-процессов с оркестрацией на LangGraph и векторной памятью Qdrant.

├── workflows/                # Бизнес-воркфлоу

## 🚀 Возможности

### 🤖 Специализированные агенты (все в папке agents/)
- **CFO (Финансовый директор)**: финансовый анализ, прибыль, налоги, денежные потоки, EBITDA, ROI
- **Главный бухгалтер**: контроль проводок, НДС, учётная политика, соответствие законодательству
- **Бухгалтер-исполнитель**: SQL-запросы к 1С, скрипты, документы, выгрузки данных
- **Экономист**: расчёт юнит-экономики, красной линии, себестоимости, рентабельности
- **Мониторинг**: отслеживание цен конкурентов, акций, рыночных трендов
- **Закупщик**: прогнозирование остатков, формирование заказов, оптимизация запасов
- **Логист**: маршруты, топливо, системы "Платон" и "Автодор", погодные условия
- **Коммерческий директор**: анализ продаж, стратегии роста, работа с клиентами, ценообразование
- **Поддержка**: обработка обращений клиентов, база знаний, решение проблем
- **SMM**: контент для соцсетей, планирование публикаций, анализ вовлечённости
- **Сайт**: аналитика, SEO, оптимизация конверсий, технический мониторинг
- **Отчётность**: формирование отчётов, Telegram-уведомления, визуализация данных

### 🧠 Технологический стек
- **LangGraph**: оркестрация агентов, условные переходы, графы выполнения
- **Qdrant**: векторная память для хранения и поиска контекста
- **Ollama**: локальные LLM модели (DeepSeek, Llama, Qwen и др.)
- **PostgreSQL**: реляционная база данных
- **Redis**: кэширование и брокер сообщений
- **FastAPI**: REST API сервер
- **Docker**: контейнеризация всех компонентов

## ⚡ Быстрый старт

### 1. Установка зависимостей
```bash
# Создание виртуального окружения
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # Linux/Mac

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Запуск инфраструктуры (Docker)
```bash
docker-compose up -d
```

Сервисы будут доступны:
- **Ollama**: http://localhost:11434
- **Qdrant**: http://localhost:6333
- **PostgreSQL**: localhost:5432
- **Redis**: localhost:6379
- **pgAdmin**: http://localhost:5050 (admin@zora.local / admin)

### 3. Загрузка моделей Ollama
```bash
# Загрузка основных моделей
ollama pull deepseek-r1:32b
ollama pull qwen2.5-coder:14b
ollama pull llama3.1:8b-instruct
ollama pull nomic-embed-text
```

### 4. Запуск ZORA
```bash
# Интерактивный режим
python main.py

# Или через модуль
python -m zora
```

## 🔧 Конфигурация

### Файл .env
Создайте `.env` файл на основе `.env.example`:
```bash
# Ollama
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=deepseek-r1:32b

# Qdrant
QDRANT_HOST=localhost
QDRANT_PORT=6333

# Базы данных
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=zora_db
POSTGRES_USER=zora
POSTGRES_PASSWORD=zora_password

REDIS_HOST=localhost
REDIS_PORT=6379

# Внешние API (опционально)
WB_API_KEY=your_wildberries_key
OZON_CLIENT_ID=your_ozon_client_id
OZON_API_KEY=your_ozon_api_key
```

### Режимы работы
В `main.py` можно выбрать режим:
- **Оркестратор**: полная цепочка агентов через LangGraph
- **Классический ZORA**: прямое взаимодействие с ядром

## 🔌 Коннекторы к внешним системам
